# COBOL Source Reference

Per-program developer documentation for every COBOL program in the
repository, generated by analyzing the actual source. Use these when you
need to know what a specific program does, what it reads/writes, what API
calls it makes (CICS, SQL, MQ, DL/I), and which copybooks and screens it
touches.

For project-wide context (architecture, directory layout, what the repo
_is_), start at [`../README.md`](../README.md).

## How to use these docs

- **Looking up one program** → grep across this folder for its name
  (`grep -rn 'COTRN02C' docs/cobol/`).
- **Tracing a transaction code** → start in `online-programs.md`, which
  has a transaction-code → program → BMS-map catalog table at the top.
- **Understanding a database access pattern** → the extension docs each
  list every `EXEC SQL`, `CALL 'CBLTDLI'`, or MQ verb call site in the
  module's programs.
- **Building a transpiler / analyzer** → each program section ends with
  a "notable patterns" subsection calling out REDEFINES, OCCURS,
  COMP/COMP-3 numerics, cursor loops, and other constructs that
  typically need special handling.

## Source-of-truth note

These docs were produced by reading the COBOL source — when the source
disagreed with module-level READMEs, the docs follow the source and flag
the discrepancy. In particular, see the queue-name and message-layout
notes in `extension-vsam-mq.md` for examples where the upstream README
was out of date relative to the code.
